Gateway API core support

After adding preliminary support last year, the ngrok Kubernetes Operator supports the entire core Gateway API. This includes:

  • Routes (HTTPRoute, TCPRoute, TLSRoute) + RouteMatches (header, path, + more)
  • Filters: header, redirect, rewrite + more
  • Backends: Backend filters + weighted balancing!
  • ReferenceGrant: RBAC for multi-tenant clusters handling
  • Traffic Policy as an extensionRef or annotation when the Gateway API isn’t flexible enough

In unique ngrok fashion, we compile all of your Gateway API definitions to Traffic Policy and accelerate them by executing those behaviors on our global cloud service.

--binding=kubernetes

Kubernetes bindings is our zero-config take on Telepresence—these endpoints that are privately accessible only inside of K8s clusters where you’ve installed the ngrok operator. Great for local development and cross-cluster networking between any two production clusters regardless of where they’re running.


# on any machine
ngrok http 8080 --url http://hello.world --binding kubernetes

# in as many k8s clusters as you'd like
helm install ngrok ngrok/ngrok-operator ...

# from a pod inside your k8s cluster
curl http://hello.world

CRDs

The ngrok operator ships with new CRDs for AgentEndpoint, CloudEndpoint, Domain + more if you want to manage ngrok resources directly without using Ingress or Gateway objects as intermediates. It’s sometimes simpler or more powerful than wrangling Ingress/Gateway objects.

The ngrok changelog: What else is new?

  • Like to live on the bleeding edge? Request early access to features like a WAF action in Traffic Policy, data residency selection, and calling external services during requests or responses.
  • Using ngrok for an open source project? You can now request a community license to better build your product and community.
  • We published a blog post on creating custom branded error pages using cloud+internal endpoints and the custom-response action.
  • You can now base your rate-limit action's bucket on any property of a request we have a variable for, giving you a degree of control and precision no other API gateway can.
  • We added tons of new IP categories to IP Intelligence for services ngrok users often want to block, like AI bots from OpenAI, Perplexity, Anthropic, and others.
  • The new isPrivateIP() macro checks whether a given IP address is part of the private address space of RFC 1918, which helps you filter IPs or set up routing without maintaining these lists yourself.
  • The jwt-validation action no longer requires an audience claim, simplifying your rules.
  • You can now see active Kubernetes Operators and Kubernetes-bound endpoints in the dashboard.
  • We're deprecating edges, and one of our first steps is preventing new users from creating edges or endpoint configurations. We've created a guide to help you migrate to their newer and better counterparts.
